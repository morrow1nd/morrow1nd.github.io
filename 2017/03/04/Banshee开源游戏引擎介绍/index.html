Banshee开源游戏引擎介绍

Posted by Morrow1nd on 2017-03-04

Banshee是高质量的现代游戏开发工具链，它是使用C++14写成的高性能多线程游戏引擎，支持2D和3D游戏开发。引擎支持Vulkan、DirectX11和OpenGL，支持跨平台输入、GUI、物理引擎、audio、动画、脚本以及其他通用的资源格式(FBX, PNG, PSD, TTF, OGG, WAV).
在引擎之上，Banshee提供直观且可定制化的编辑器，便于美术、设计师、程序的使用。编辑器允许你管理整个项目的资源、脚本、测试和发布游戏。编辑器允许开发者通过脚本扩充功能。

以上翻译自官方的介绍，看的云里雾里的。对于Banshee是一个什么样的游戏引擎，我觉得一句话可以说明白：高仿Unity。

是的，Banshee为开发者提供的工作流程和Unity几乎一模一样。Unity和Banshee的核心概念是一样的。

功能

下面的总结取自Banshee官方网站。

  • Editor
    • All-in-one toolset
    • 在编辑器中运行游戏
    • Asset 热更新
    • Automatic GUI for custom script objects(脚本中的变量会自动出现在Inspector中)
    • 一键构建游戏
    • Extension API
  • C# scripting
    • C# 6.0
    • Visual Studio集成
    • API文档
  • 现代c++14代码
    • 提供Low-level framework API (Unity没有的)
    • Fully documented
  • 渲染
    • 多线程渲染
    • DirectX 11, OpenGL, Vulkan
  • Animation
  • Audio
  • Asset pipeline
  • 物理引擎
  • GUI

引擎架构

整个引擎分四层: Utility, Core, Engine, Editor。Utility层提供跨平台的线程、文件系统存取、事件机制、数学库、c++反射。Core层提供整个引擎各模块的接口（虚基类），供上层调用。Engine层使用Core层的接口实现一些基本的功能，如GUI、脚本。Editor层实现和编辑器相关的功能，Editor层的主要部分是使用C#脚本实现的。具体细节见官方文档。

除了以上四层架构外，还有一项很重要的设计：插件。Core层的接口都是在插件中实现的，上图中的黑色块就是插件。

开发进度

Banshee的源码托管在GitHub上，主要开发者只有原作者一个人。目前正在添加PBR的功能。Banshee到现在只发布了v0.4，预计在2017年推出1.0版本。Banshee的源码不稳定，各种Bug层出不穷，目前想直接使用Banshee开发游戏是不太可能的。

总结

Banshee吸引我的不是它能做什么，而是它里面的设计思想，Banshee有大量的文档讲述底层的实现，我可以透过Banshee看到Unity底层的细节。以前我寻找了很多的开源引擎，比如Godot, Urho3D等等，但它们总让我感到有些别扭。

  • Godot - 有关内部设计的文档几乎没有
  • Urho3D - 编辑器老旧
  • Torque3d - 编译不方便，文档不清

以上这些引擎都有稳定的发行版，有使用的案例，有一定数量的用户，这些Banshee都没有。

但Unity有。
